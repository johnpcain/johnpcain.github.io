Why, hello. Thanks for visiting my blog! Today we're going to discuss positioning elements on a web page. A web page is made up of a host of different things - pictures, text, boxes, graphs, etc. In coding, we refer to those different things as elements. When we are coding the web page, we use CSS to tell the web page where to place those different elements. We have several ways of telling the web page how to order these elements, which are listed and discussed below. Let's say we have three elements on a page - block 1, block 2, and block 3. Throughout this blog post, i'll be using different types of positioning in CSS to show how we can arrange these different blocks.

Different types of Positioning

  • Static
  • Fixed
  • Absolute
  • Relative

Static

Static positioning is the default positioning that is applied when no positioning is indicated in the css stylesheet. Elements will be ordered sequentially based on the order in which they were coded in html. I've told CSS to treat each block as a block using the code "display:block" because... well because they're blocks. Below, you can see that the default positioning applies by placing the first block in the top left corner of the web page and every subsequent block directly below it.

Fixed

Fixed positioning tells the element "stay where you are on the screen, regardless if the user scrolls to a different part of the web page. Fixed positioning is often helpful for directories that should always be made available to the user. Another interesting little tidbit about fixed positioning is that the other static elements are now default positioned to the web page - you can see that block 2 now occupies that top left spot in the web page.

Absolute

Abosolute positioning is absolutely positionined relative to the parent element's positioning. If there is no parent element, then the element will be positioned relative to the web page. In our example below, I coded the elements as "siblings", meaning that none of the elemnts have parents. It's sad, I know.

As you can see above, the Block 1 element was positioned absolutely, while the other two remained static. Similar to fixed positioning, the next element in line after the absolute element will occupy the top left space on the web page.


Now below, i've given Blocks 1 & 2 the same absolute positioning, but made Block 1 the proud parent of Block 2. As you can see, Block 1 is positioned relative to the web page, and Block 2 is positioned relative to Block 1. So kewl.

Relative

Elements that are positioned using relative tags are positioned relative to the space they would have occupied, had default positioning applied. In the example below, i've given Block 1 relative positioning and left the other two static. Block 1 now is positioned relative to the top left corner, but you'll also notice that Blocks 1 & 2 still default position relative to the space that Block 1 would have occupied. Blocks 1 & 2 are respecting Block 1's territory, so to speak.

Now, there is one last point i'd like to illustrate. Whenever one element is given absolute, fixed, or relative positioning, it automatically receives a z-index value. Think of z-index value as street cred. Elements that are not applied any positioning and receive static positioning do not have a z-index value. So, when two elements are fighting for the same space on a web page, the element with the higher street cred occupies the space.
