What is the method "group_by"?

The group_by method falls under the module "enumberator" because it can only be called on a collection of code (think arrays, hashes, etc.). The method, in essence, groups the elements in the collection of code together according to the code that is outlined in the syntax of the method. The syntax of the .group_by method is .group_by {|object| block}, where the object merely names the variable to be used in the block, and the block sets forth the code that is used to group the elements in the collection.

The group_by method returns a hash as its output, where the keys are all the grouped results from the block of code, and the values are the arrays of elements that correspond to the key. The elements in the collection can be arranged and grouped in any which way, to name a few: element length (4 letter words vs. 5 letter words), element frequency (finding the mode!), or element category ( cats vs. dogs).

When is this method used?

This method is very useful when trying to draw conclusions from large groups of data. Say, for instance, an array held the results of a student election, where the students were asked to submit a name for one of four candidates. We're telling the new hash to associate every key ( or candidate) with the size of the array, instead of the array itself.
