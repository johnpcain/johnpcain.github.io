Class Methods vs. Instance Methods

I chose to discuss this topic because I was, at first, pretty intimidated by the use of "self." and didn't quite understand why one would choose to use one method over the other.

In ruby-speak, class methods are only available on the class itself, whereas instance methods are only available on instances of the class. To bring it down to earth a bit, think of classes as humans, and instances as individual people (you and me). Let's say we wanted to calculate the height of Jim Carrey. We wouldn't want to call a .height method on Humans, because the options are limitless. Instead, we would want to call the .height method on an instance of a Human. Human.new(Jim_Carrey).height

No, height isn't a real method, but I think we both get the point. Generally speaking, it's best practice to use instance methods when writing code that involves logic or operations. Class methods are used when you don't have an instance available to you, or for things that don't operate on an individual instance of a model.
